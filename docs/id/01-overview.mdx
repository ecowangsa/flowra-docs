# Gambaran Umum & Filosofi Flowra

## Apa itu Flowra?
Flowra adalah framework Node.js modular yang membantu tim merilis API siap produksi tanpa kehilangan kendali atas Express. Flowra menambahkan lapisan inti yang ramping namun terarah di atas server HTTP yang sudah familiar, membungkus routing, dependency injection, konfigurasi, dan otomasi CLI ke dalam satu sistem terpadu. Setiap aplikasi Flowra terdiri dari modul-modul eksplisit yang menyatukan route, controller, service, dan akses data sehingga engineer baru bisa memahami alur fitur secara cepat.

## Filosofi Inti
### Komposisi berbasis flow
Flowra mendorong Anda mendeskripsikan penanganan request end-to-end sebagai sebuah flow. Setiap modul memiliki entry point ringkas yang merangkai middleware, logika controller, dan service hilir. Hal ini membuat jalur request menjadi jelas serta memudahkan reasoning terhadap perilaku lintas modul seperti logging atau translasi error.

### Modul yang eksplisit
Alih-alih menyebar file ke berbagai folder ad-hoc, Flowra mengelompokkan seluruh berkas terkait—kontrak HTTP, validasi, logika bisnis, hingga persistensi—di dalam `app/modules/<context>`. Modul tetap ringan sehingga membuat bounded context baru cukup menjalankan generator CLI.

### Service container sebagai pusat
Jantung Flowra adalah service container yang didefinisikan di `core/container/application-container.ts`. Container ini mendaftarkan singleton seperti HTTP server, repository konfigurasi, logger, koneksi Knex, serta service buatan pengguna. Controller dan service lain mendeklarasikan dependensi yang dibutuhkan, kemudian Flowra akan me-resolve-nya, meningkatkan kemampuan testing dan memunculkan grafik dependensi yang jelas.

### CLI deklaratif
CLI Flowra (`bin/flowra.ts`) memahami konvensi framework dan dapat membuat modul, controller, service, migrasi, maupun test. Resep tersimpan dalam template deklaratif sehingga tim bisa mengabadikan best practice dan menjaga konsistensi repositori.

## Mengapa Flowra dibanding Express murni?
Express sengaja minim opini, sehingga setiap tim harus terus-menerus menciptakan ulang batas modul, dependency injection, pemuatan konfigurasi, serta otomasi CLI. Flowra memecahkan masalah berulang tersebut dengan menyediakan:

- **Struktur yang prediktif** lewat tata letak modul di `app/modules` serta entry point `main/bootstrap.ts`.
- **Dependency injection bawaan** melalui service container sehingga wiring manual hilang dan pengujian berbasis konstruktor lebih mudah.
- **Konfigurasi terpadu** yang ditangani `core/config`, termasuk default sesuai environment dan akses bertipe.
- **Otomasi CLI** yang menjaga konsistensi proyek, dari `flowra create-app` hingga `flowra make:controller users/list`.
- **Ergonomi testing** berkat helper di `core/testing` yang menyalakan container secara terisolasi.

## Demo end-to-end
Membuat dan menjalankan aplikasi Flowra hanya butuh satu menit.

```bash
npx flowra create-app my-api
cd my-api
npm install
npx flowra make:module health
npx flowra serve
```

Setelah server berjalan Anda bisa mengakses `http://localhost:3000/health`:

```bash
curl http://localhost:3000/health
# => {"status":"ok"}
```

Flowra memantau berkas TypeScript Anda dan melakukan reload otomatis sambil mempertahankan isi container.

## Kapan sebaiknya memakai Flowra?
Pilih Flowra ketika Anda membutuhkan sesuatu yang lebih dari sekadar routing library namun tidak ingin framework monolitik. Skenario ideal meliputi:

- Tim yang bermigrasi dari aplikasi Express kustom dan ingin struktur tanpa kehilangan fleksibilitas.
- Produk yang harus meng-onboard engineer baru dengan cepat sekaligus menegakkan batas modul konsisten.
- Organisasi yang mengutamakan testability, dependensi eksplisit, serta workflow deklaratif.
- Startup yang akan menumbuhkan fitur secara agresif dan memerlukan CLI untuk menjaga keseragaman modul.

Jika kebutuhan Anda hanya endpoint mikro atau proxy ringan, Express murni mungkin lebih sederhana. Untuk kasus yang lebih kompleks, desain modular Flowra menjaga backend besar tetap mudah dirawat.
