# Pertanyaan yang Sering Diajukan

## Apa perbedaan Flowra dengan Express?
Flowra menyediakan struktur modular, service container, dan scaffolding CLI di atas Express. Anda tetap mendapatkan fleksibilitas Express tanpa wiring manual. Lihat [Gambaran Umum & Filosofi](/docs/id/overview).

## Bisakah saya memakai JavaScript alih-alih TypeScript?
Bisa. CLI secara default menghasilkan TypeScript, namun Anda dapat mengatur `flowra.config.ts` untuk mengeluarkan JavaScript dan menyesuaikan `tsconfig.json`. Simpan definisi tipe demi tooling yang lebih baik meskipun kode runtime JS.

## Apakah Flowra mendukung WebSocket?
Fokus utama Flowra adalah HTTP. Anda bisa mengintegrasikan Socket.IO dengan mendaftarkannya sebagai service di container dan memasangnya di `main/bootstrap.ts`. Pastikan namespace berada di dalam modul agar batas tetap terjaga.

## Bagaimana menjalankan migrasi di CI?
Set `DB_MIGRATE=true` dan jalankan `npx flowra migrate` sebelum aplikasi dinyalakan. Kombinasikan dengan `npm run build` untuk memastikan kompilasi TypeScript. Lihat [Konfigurasi Knex](/docs/id/data/knex-setup).

## Framework testing apa yang sebaiknya digunakan?
Jest sudah dikonfigurasi, namun Anda bisa memakai Vitest atau Mocha. Pastikan runner memanggil `core/testing/application-test-module.ts` agar modul termuat benar. Rujuk [Ringkasan Pengujian](/docs/id/testing/overview).

## Bisakah saya berbagi service antar modul?
Bisa. Daftarkan service di `app/providers/index.ts` atau provider modul masing-masing. Inject melalui container memakai dekorator `@inject()`. Lihat [Service container](/docs/id/architecture/service-container).

## Bagaimana menangani background job?
Gunakan provider antrian di `core/queues`. Daftarkan queue BullMQ dan buat worker di `app/modules/<context>/workers`. Contoh lengkap tersedia pada [Resep](/docs/id/recipes).

## Apakah Flowra siap produksi?
Ya. Flowra digunakan di layanan produksi dan menyertakan best practice untuk logging, migrasi, serta deployment. Tinjau [Panduan Deployment](/docs/id/deployment/guide) dan [Ringkasan Keamanan](/docs/id/security/overview) sebelum rilis.
