# Installation & Setup

## Prerequisites
| Requirement | Version | Notes |
| ----------- | ------- | ----- |
| Node.js | >= 20.0 | Flowra targets modern JavaScript features and native fetch. |
| npm | >= 10.0 | Yarn and pnpm work as well; commands below use npm. |
| OS | macOS, Linux, Windows | CLI tested on popular shells (bash, zsh, PowerShell). |

> **tip**
> Enable TypeScript incremental builds (`tsc --watch`) for faster feedback while using Flowra's CLI generators.

## Install the CLI
You can install Flowra globally or run it through `npx` when needed.

```bash
npm install -g flowra
flowra --help
```

For reproducible environments—CI pipelines or multi-developer projects—prefer a local dev dependency:

```bash
npm install --save-dev flowra
npx flowra --help
```

## Scaffold a new application
The quickest way to start is the `create-app` command. It pulls the template from the Flowra source tree and sets up TypeScript, linting, and testing defaults.

```bash
npx flowra create-app retail-api
cd retail-api
npm install
```

The generator bootstraps these files:

```text
retail-api/
├─ app/
│  ├─ modules/
│  │  └─ example/
│  │     ├─ http/
│  │     │  ├─ controllers/
│  │     │  └─ routes.ts
│  │     ├─ services/
│  │     └─ validators/
├─ bin/
│  └─ flowra.ts
├─ core/
│  ├─ container/
│  ├─ config/
│  ├─ http/
│  ├─ knex/
│  └─ testing/
├─ main/
│  └─ bootstrap.ts
├─ public/
├─ resources/
│  └─ views/
├─ orm.cli.config.js
├─ tsconfig.json
└─ package.json
```

## Configure environment variables
Flowra reads `.env` during bootstrap (`core/config/env.ts`). Copy the example file and adjust secrets per environment:

```bash
cp .env.example .env
```

Key settings include database credentials (`DB_CONNECTION`, `DB_HOST`, `DB_USER`), HTTP server port (`APP_PORT`), and logging level (`LOG_LEVEL`).

## 10-minute quickstart checklist
1. ✅ Install dependencies with `npm install`.
2. ✅ Duplicate `.env.example` into `.env` and fill connection details.
3. ✅ Run `npx flowra make:module health` to create your first module.
4. ✅ Implement `app/modules/health/http/controllers/health.controller.ts` to return `{status:'ok'}`.
5. ✅ Register the HTTP route in `app/modules/health/http/routes.ts`.
6. ✅ Start the dev server with `npx flowra serve`.
7. ✅ Test the endpoint via `curl http://localhost:3000/health`.
8. ✅ Add a user service using `npx flowra make:service users/create-user`.
9. ✅ Wire a Knex migration through `npx flowra make:migration create_users_table` and run `npx flowra migrate`.
10. ✅ Commit the generated scaffold and share the onboarding guide with your team.
